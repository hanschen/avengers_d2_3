Here,
we report on a new ensemble-based dual-state CO2 flux and concentration data assimilation method
developed within the TRACE Regional Atmosphere--Carbon Ensemble (TRACE) framework.
One of the goals of this inversion approach is to enable the assimilation of high volumes of dense satellite observations.
Thus,
we decided to focus on observations from the CO2M satellite,
which provides much more data than existing satellites such as OCO-2.
However,
as the launch date of the first CO2M satellite was delayed,
we chose to use synthetic observations instead.
This allows for more in-depth validation of the experiment results as,
unlike in the real world,
the truth is known.
With that said,
it is important to emphasize that the results here are idealistic and do not reflect the real world.


\section{Method}

The method is based on the ensemble-based simultaneous state and parameter estimation (ESSPE) approach for flux inversion
introduced in Chen et al. (2023).  % TODO
Figure~\ref{fig:trace} shows a simplified schematic of approach implement within the TRACE framework.
TRACE uses different flux models to simulate the fluxes from different sources based on a set of parameters.
In the current implementation,
the parameters are simple scaling factors applied to the prior fluxes,
similar to many other inversion systems such as CarbonTracker (CITATION).
In the future it would be possible to implement more sophisticated forward models for fluxes similar to CCFFDAS
(Section X).

Using an ensemble of perturbed flux parameters,
the system first simulates an ensemble of fluxes.
These fluxes are used as input for a regional atmospheric transport model,
which computes the resulting CO2 concentrations based on fluxes,
initial conditions,
and lateral boundary conditions.
The initial and lateral boundary conditions can include different CO2 concentrations but also different meteorological states
to simulate atmospheric transport errors (Chen et al., 2019).  % TODO
Unlike many traditional inversion systems such as LUMIA (Section x),
TRACE uses an online mesoscale weather model to downscale the meteorological conditions from a coarses dataset to higher spatial and temporal resolutions.

The resulting simulated CO2 concentrations from the atmospheric model and prior ensemble of flux parameters
are then updated based on CO2 concentration observations in the data assimilation component.
The data assimilation scheme updates both the CO2 concentrations and flux parameters.
The rationale behind this dual-state approach is that an update of a flux parameter will also result in changes in the simulated CO2 concentrations.
By updating both simultaneousy based on ensemble correlations,
there is no need to re-run the atmospheric transport model after updating the parameters.

Here we provide a summary of the data assimilation approach;
for further details,
we refer to Chen et al. (2023).  % TODO
The approach implemented in TRACE is based on the ensemble Kalman Filter (EnKF).
In the standard Kalman update,
the background (prior) state vector $\mathbf{x}^b$ is updated based on observations $\mathbf{y}^o$
to yield the analysis (posterior) state vector $\mathbf{x}^a$:
\begin{equation}
    \label{eq:enkf_update}
    \mathbf{x}^a = \mathbf{x}^b + \mathbf{K}(\mathbf{y}^o - \mathbf{H}(\mathbf{x}^b)),
\end{equation}
where $\mathbf{H}$ is the linear observation operator relating model variables to observations,
and $\mathbf{K}$ is the Kalman gain matrix,
computed as:
\begin{equation}
    \label{eq:kalman_gain}
    \mathbf{K} = \mathbf{P}^b \mathbf{H}^\top (\mathbf{H}\mathbf{P}^b \mathbf{H}^\top + \mathbf{R})^{-1}.
\end{equation}
Here,
$\mathbf{P}^b$ is the error covariance matrix for the background (prior) and
$\mathbf{R}$ is the error covariance matrix for the observations.

After assimilating the observations the prior error covariance matrix is updated according to:
\begin{equation}
    \label{eq:posterior_cov}
    \mathbf{P}^a = (\mathbf{I} - \mathbf{K}\mathbf{H})\mathbf{P}^b,
\end{equation}
where $\mathbf{P}^a$ is the posterior (analysis) error covariance matrix,
and $\mathbf{I}$ is the identity matrix.

Unlike traditional atmospheric inversion approaches,
which include only fluxes or flux parameters in the state vector,
here $\mathbf{x}$ consists of both gridded two-dimensional flux parameters and three-dimensional CO2 concentrations.
Propagating the full background error covariance matrix for such high-dimensional systems using the dynamical model is typically not computationally feasible.
Another computational challenge is to perform the matrix inversion in Equation~\ref{eq:kalman_gain} when the number of assimilated observations is large,
which is often case for satellite observations.
To address these challenges,
the EnKF algorithm approximates the full background error covariance matrix by an ensemble approximation:
\begin{equation}
    \label{eq:ensemble}
    \mathbf{P}^b = \frac{1}{N - 1} \mathbf{X}' \mathbf{X}'^\top,
\end{equation}
where $N$ is the number of ensemble members
and $\mathbf{X}'$ is a matrix whose columns are state vectors of the ensemble perturbations (deviations from the ensemble mean).
Instead of propagating the full $\mathbf{P}^b$,
each ensemble member is propagated forward in time using the dynamic model.

Additionally,
TRACE uses a deterministic square-root formulation of the EnKF,
in which observations are assimilated serially;
each time an observation is assimilated,
the ensemble mean is updated according to Equation~(\ref{eq:enkf_update}).
At the same time,
the ensemble perturbations are updated to reflect the reduced uncertainty:
\begin{equation}
    \label{eq:mem_update}
    \mathbf{x}'^a = \mathbf{x}'^b - \widetilde{\mathbf{K}}\mathbf{H}\mathbf{x}'^b,
\end{equation}
where $\mathbf{x}'^a$ and $\mathbf{x}'^b$ are the posterior and prior ensemble perturbations,
respectively,
and $\widetilde{\mathbf{K}}$ is the reduced Kalman gain:
\begin{equation}
    \label{eq:reduced_kalman_gain}
    \widetilde{\mathbf{K}} = \alpha\mathbf{K}, \quad
    \alpha = \left(1 + \sqrt{\frac{\mathbf{R}}{\mathbf{H}\mathbf{P}^b\mathbf{H}^\top + \mathbf{R}}} \right)^{-1}.
\end{equation}

Another advantage of the EnKF implementation is that it allows for the use of the full (potentially nonlinear) observation operator $h$,
which is applied to each ensemble member to map model states to observations.

EnKF applications typically require techniques to address issues such as rank-deficient and spurious correlations in $\mathbf{P}^b$
and filter divergence due to underestimated posterior errors.
Here,
we applied covariance localization,
relaxation to prior perturbations,
and relaxation to initial perturbations for the flux parameters.
See \citet{chen2023} and \citet{houtekamer2016} for detailed information about these techniques.

Atmospheric inversions typically use an assimilation window to account for the time lag for CO2 fluxes to influence CO2 concentration observations.
Such a time window is not strictly necessary in the dual-state approach;
the relationship between CO2 fluxes and concentrations are preserved in $\mathbf{P}^b$,
as the state vector includes both fluxes/flux parameters and concentrations.
\citet{chen2023} shows an example of how a filtering approach without an assimilation window still is able to constrain fluxes in remote locations
(see their Figure 5).
Including an assimilation window can however be helpful to average out large random errors.
In this deliverable,
we chose to not include an assimilation window to get a clearer picture of the impact of each assimilation cycle.


\section{Setup}

We performed two OSSEs assimilating synthetic XCO2 observations from a hypothetical CO2M satellite:
one in summer and another in winter,
constraining fluxes in July and February 2021,
respectively.
We focused on biospheric fluxes,
as these tend to produce the largest signals in atmospheric CO2 concentrations at regional scales.


\subsection{Data}

We used meteorological initial and boundary conditions from the ERA5 reanalysis (CITATION) to drive the regional atmospheric model.
Hourly ERA5 data were obtained on a 0.25\degree{} resolution grid on 37 pressure levels.

For atmospheric initial and lateral boundary conditions in CO2 concentrations,
we used the CAMS global greenhouse gas inversion constrained by surface observations.
The CAMS data were provided on a 1\degree{} grid,
interpolated from the hexagon grid ($\sim$90 km resolution) of the transport model.

The prior biospheric fluxes were taken from an LPJ-GUESS simulation (0.5\degree{} resolution) downscaled to hourly values,
see Section X for more details.
Fluxes from the ocean,
fossil fuel emissions, and
fires
were assumed to be perfectly known.
As such,
they were not included in the experiments,
as their impacts would be just subtracted out in the data assimilation procedure.


\subsection{XCO2 observations}

The locations of XCO2 observations were derived from a dataset of synthetic CO2M observations prepared by XX for the CoCO2 project.
The satellite retrievals were averaged to superobservations at the same resolution as the atmospheric transport model (about 25 km).
Only observations over land (nadir mode) were used.
Figure~\ref{fig:xco2} shows the observation count for each model grid cell for the two experiments.

\begin{figure}[tb]
    \centering
    \includegraphics[width=\textwidth]{figures/xco2.png}
    \caption{Number of synthetic XCO2 observations from a hypothetical CO2M satellite in
        (a) summer and
        (b) winter 2021
        that were assimilated in the experiments.
        The XCO2 observations were aggregated from their original resolution (about 2 km)
        to superobservations at the atmospheric transport model grid (about 25 km resolution).
    }
    \label{fig:xco2}
\end{figure}

The XCO2 observation values were derived from the truth simulation using the mass-weighted vertical integral of CO2 concentrations \citep{FIXME}.
Random errors were sampled from a normal distribution with a standard deviation of 1 ppm and were added to all XCO2 observations.
Here we did not include systematic errors;
this aspect will be further explored in WP5.

For each XCO2 superobservation,
we calculated the fraction of grid cell covered by observations,
taking also into account cloud cover.
The cloud cover was taken from the truth simulation.
To convert vertically resolved cloud cover from the atmospheric model to total cloud cover in each grid cell,
we assumed exponential--random overlap \citep{FIXME} with a vertical decorrelation length scale of 2 km.
Only superobservations in grid cells with over 90\% observation coverage were retained for assimilation.
